# Security Policy

## Supported Versions

Security fixes are applied to the latest stable release on `master`.

## Reporting a Vulnerability

If you discover a security issue, please report it privately before public disclosure.

- Open a private security advisory on GitHub (preferred), or
- Contact the maintainer through repository contact channels.

Please include:

- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested remediation (if available)

We aim to acknowledge reports quickly and provide a remediation plan.

## Responsible Use

This project is intended for defensive security analysis, reputation checks,
and incident-response support.

Do **not** use this software for unauthorized access, disruption, fraud, or
other unlawful activity. You are solely responsible for how you use this tool.
